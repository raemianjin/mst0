# myStt v1.0.0

대용량 음성/동영상을 **조각(chunk) 단위로 나눠 STT** 한 뒤 합쳐서 돌려주는 안드로이드 앱.
초대용량 동영상에서 **음원만 추출**하는 기능도 포함. (made by 영찬영하 Daddy / J패밀리)

---

## 왜 기존 STT 는 큰 파일에서 실패했나

참고로 받은 MyMeeting 구조의 두 가지 한계:

1. **디코딩 OOM** — 파일 전체 PCM 을 메모리(ByteArrayOutputStream) 한 곳에 올림.
   큰 파일이면 수백 MB~GB 가 되어 앱이 죽음.
2. **업로드 한도** — 클라우드 STT 에 파일 전체를 한 번에 업로드.
   Groq Whisper 의 ~25MB / 길이 제한에 걸려 실패.

## myStt 가 푸는 방식 (핵심)

```
[원본 오디오/동영상]
      │  MediaExtractor + MediaCodec 로 오디오 트랙만 스트리밍 디코딩
      ▼
[무음 경계에서 조각내기]   ← 목표 길이(기본 120초) 부근에서 가장 조용한 지점을 찾아 절단
      │                      → 단어 중간이 잘리지 않음
      ▼
[16kHz mono WAV 청크]  → 디스크에 1개씩 기록 (메모리는 청크 1개 분량만 점유)
      │  Channel(용량 제한) 로 백프레셔 → 디스크에 무한정 안 쌓임
      ▼
[병렬 STT 소비자 N개]   → 각 청크를 Groq Whisper 로 인식, 끝나면 WAV 즉시 삭제
      │                   → 429/5xx 는 지수 백오프 재시도(최대 6회)
      ▼
[순서대로 병합] → .txt 저장
```

- **메모리 한정**: 파일이 10GB든 100GB든 동시에 점유하는 메모리는 청크 1개(수 MB)뿐.
- **부분 실패 허용**: 한 조각이 끝내 실패해도 전체 중단 없이 `[조각 N 인식 실패]` 표식만 남기고 나머지는 보존.
- **취소/중단 안전**: 도중 취소해도 그때까지 인식분을 `_부분.txt` 로 저장.
- **실시간 미리보기**: 앞쪽 조각부터 인식되는 대로 화면에 누적 표시.

## 동영상 → 음원 추출

- 1순위 **무손실 트랙 복사**(재인코딩 없음) → 초대용량도 수 초, 음질 그대로.
- 코덱이 안 맞으면 **디코드→AAC 재인코딩** 폴백.
- 출력은 `.m4a`(AAC).
  > ⚠️ 솔직한 한계: 안드로이드 표준 인코더에는 **MP3 인코더가 없습니다.**
  > 그래서 `.mp3` 직접 생성은 못 합니다. `.m4a` 는 모든 플레이어에서 재생되고
  > 같은 음질에 용량이 더 작습니다. 꼭 진짜 `.mp3` 가 필요하면 LAME(NDK) 인코더를
  > 넣는 후속 작업이 필요합니다(빌드 안정성 때문에 이번엔 제외).

## 사용법

1. **설정**에서 Groq API 키 입력 (console.groq.com 무료 발급). 키는 기기에 암호화 저장.
2. 홈에서 모드 선택:
   - **음성 → 텍스트**: 오디오/동영상 선택 후 `STT 시작`.
   - **동영상 → 음원**: 동영상 선택 후 `음원 추출`.
3. 결과는 복사/공유/`.txt` 저장 가능. STT 결과는 `회의록`/`요약` AI 정리도 가능(선택).

### 설정 항목
- **조각 길이**: 60 / 120(권장) / 300초. 길수록 호출 수↓, 실패 시 손실↑.
- **동시 처리 수**: 1(권장) ~ 3. 높이면 빠르지만 Groq 무료 한도(429)에 걸리기 쉬움.
- **모델**: Turbo(권장·빠름) / Large v3(정확·느림).
- **언어**: 한국어 / English / 日本語.
